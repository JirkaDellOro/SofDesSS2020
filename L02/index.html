Framework vs Library

Dass die beiden Begriffe synonym verwendet werden liegt vermutlich daran, dass sie das selbe Ziel verfolgen: Sie erweitern den Funktionsumfang.

Library

Dabei handelt es sich um eine Sammlung von Klassen und Funktionen. Der Nutzer der Library hat dabei die volle Kontrolle wann und in welchen Umfang er die nötigen Funktionen aufruft. Wichtig zu erwähnen: Die Funktionalität der Library ist abgeschlossen und muss nicht vom Nutzer erweitert werden.

Beispiele:

  • React.js
  • GSAP/GreenSock
  • JQuery

Framework

Der letzte Satz zeigt somit schon den wichtigsten Unterschied der beiden Begriffe. Ein Framework muss für die speziellen Anforderungen des Nutzers noch angepasst werden. Der Entwickler gibt einen gewissen Teil der Kontrolle ab. Meistens wird schon eine bestimmte Ordnerstruktur angelegt.

Beispiele:

  • Angular
  • Vue.js
