Clean Code

Naming

Der Autor bezeichnet die Benennung von Variablen, Parametern und Methoden als den wichtigsten Aspekt des Clean Coding. In der Lektion 03 wird es nur als einer der wichtigsten Aspekte benannt. Auch er benutzt die lowerCamelCase Methode. Bei den Parametern setzt er nicht auf einen Suffix wie "ss_variable" oder "_variable". Methoden sind bei ihm auch lowerCamelCase und bezeichnen mit Ihrem Namen die Funktionalität der Methode.

Magic numbers

Im Vergleich zum Skript hält der Autor viel davon feste Zahlenwerte die einen Status ausdrücken (Bsp. NotRegistered = 1) in Form eines Enums zu definieren. Im Skript wird "nur" dazu geraten die solche Variablen entsprechend selbsterklärend zu benennen.

More Readable

Der Autor des Artikels rät dazu, wenn möglich if-else if - Statements durch entsprechende switch-cases zu ersetzen. Außerdem sollte man zu große Zuweisungen in Variablen zwischenspeichern. So ist der Code einfacher zu lesen.

Beispiel

                

                    //Schlecht
                    priceAfterDiscount = (price - (0.5m * price)) - (discountForLoyaltyInPercentage * (price - (0.5m * price)));

                    //Gut
                    priceAfterDiscount = (price - (0.5m * price));
                    priceAfterDiscount = priceAfterDiscount - (discountForLoyaltyInPercentage * priceAfterDiscount);

Output

Erzeugt für folgende Klasse Person eine eigene ToString() Methode. Erzeugt eine Test-Routine, in einem Array fünf Instanzen von Person erzeugt und in einer Schleife alle Personen auf der Konsole ausgibt, die älter als 20 sind.

Not obvious bug

Der Autor des Artikels nimmt hier bezug auf Variablen, die durch Zahlenwerte bestimmte Status darstellen. Für den Fall, dass eine solche Variable einen Wert annimmt den wir nicht definert haben. Soll eine entsprechende Exception geworfen werden. Darauf legt der Autor des Skripts nicht so viel Wert.

Don't repeat yourself

Sowohl im Skript als auch im Artikel wird empfohlen Code-Schnipsel, die sich wiederholen in eigenen Methoden zu verpacken.
